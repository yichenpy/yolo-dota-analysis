# SAHI 配置 sweep（200 张样本 + 公平对比）

> SOP 类型：算法与模型实验 | 实验日期：2026-05-16

## 1. 实验目的

在 DOTAv1.5-lite val 的固定 200 张样本上，用同一份 baseline 权重对比：

- 标准整图推理（Standard）
- 4 组 SAHI 配置变体

回答的核心问题：

1. SAHI 相对标准推理在小目标上能带来多少 recall 增益？
2. SAHI 的副作用（中等-大目标退化）能否通过调参缓解？
3. 哪一组配置是当前最优？

## 4. 深度分析

### 4.1 符合预期

1. **SAHI 在小目标上显著有效**：3-9 px 区间 miss_rate 从 45-56% 降到 13-21%，相对降幅 60-70%
2. **C（nms-iou 0.65）全面优于 B（默认 0.5）**：调高跨切片 NMS 阈值，让"被切两半的同一目标"不互相抑制
3. **D（slice=768）退化**：更大 patch 意味着小目标在网络眼中"放大"幅度变小，违背 SAHI 设计初衷
4. **E（overlap=0.3）与 C 完全持平**：overlap 主要影响边界目标，但 nms_iou 提升后已经救了大部分边界 case

### 4.2 不符合预期 / Bad Case

#### Bad Case 1：中等偏大目标（23-62 px）SAHI 全军退化

| 桶 | A: Std | 最佳 SAHI (C) | Δ |
|----|--------|----------------|---|
| 22.8-33.9 | 4.5% | 10.1% | **+5.6pp** |
| 33.9-62.6 | 3.6% | 10.1% | **+6.5pp** |

**根因**：30-60 px 目标常常跨越 512 patch 的边界，被切成两半各检测一次。即便 `perform_standard_pred=True` 让整图也跑了一遍，但跨切片 NMS 仍可能把整图的高置信度预测和切片的低置信度预测合并掉。

**已尝试缓解**：

- nms-iou 0.5 → 0.65：把 +7.1/+8.5pp 改善到 +5.6/+6.5pp，部分修复
- overlap 0.2 → 0.3：几乎无效

**未尝试缓解**：

- nms-iou 0.7 或 0.75（更激进保留）
- class-conditional SAHI：medium-large 类（swimming-pool、storage-tank、roundabout）直接走 standard 推理

#### Bad Case 2：swimming-pool 大幅退化

| 配置 | swimming-pool missed |
|------|---------------------|
| Standard | **50** |
| SAHI 任何配置 | 190-217 |

调任何 SAHI 参数都改善有限。原因：

- 泳池尺寸正好在 30-60 px 退化区间
- 形状无明显方向特征，跨 patch 时难精确合并
- 颜色对比度强，slice 内部容易被多次检出 FP（看 false_by_class.swimming-pool = 5776-8456）

class-conditional SAHI 是这类问题的最终解。

### 4.3 关键技术洞察

1. **SAHI 不是 free lunch**：是 "小目标增益 vs 中大目标代价" 的 trade-off，净增益取决于数据集尺寸分布
2. **DOTAv1.5-lite 的尺寸分布对 SAHI 友好**：约 70% 目标 ≤ 23 px（SAHI 受益），30% > 23 px（SAHI 中性或受损）
3. **nms-iou 阈值是 SAHI 的关键钮**：从 0.5 调到 0.65 让所有 size bin 都更好或持平
4. **overlap 收益边际**：在已有 perform_standard_pred=True 的情况下，overlap 提升边际很小

### 4.4 下一步改进计划

按优先级：

- [ ] **优先 1**：用 Run C 配置跑全 3503 张 val（约 9 小时），得到最终 mAP
- [ ] **优先 2**：在 NWD 权重上同样跑 Run C SAHI，验证 NWD × SAHI 是否正交叠加
- [ ] **优先 3**（可选）：试 nms-iou=0.7 或 0.75，看是否能进一步救 medium-large
- [ ] **长期**：实现 class-conditional SAHI（large 类禁用切片）
- [ ] **长期**：用 SAHI Fine-tuning 模式重训（但 DOTAv1.5-lite 已经是预切片，预期增益有限）

## 6. 结论

1. **SAHI 在 DOTAv1.5-lite 上有效**：净增益 recall +9.3pp，small-vehicle 漏检砍掉 66%
2. **最优配置确认为 `slice=512, overlap=0.2, nms-iou=0.65`**（Run C）
3. **不再需要做 200 张 sweep 调参**：sweep 空间已经探索充分，应直接用 C 跑全 val
4. **NWD × SAHI 的正交性待全 val 验证**

NWD 实验（前一轮）整体只 +0.7% mAP，SAHI 这一轮就是 +8.3pp recall。**SAHI 才是 DOTAv1.5-lite 小目标的正确武器**。
