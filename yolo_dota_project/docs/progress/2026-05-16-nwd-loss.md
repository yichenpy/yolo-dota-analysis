      --name yolo11s_baseline_short --epochs 20 --batch 8

### 4.3 关键决策记录

| 决策 | 理由 | 影响 |
|------|------|------|
| 用 monkey-patch 而非改 ultralytics 源码 | ultralytics 升级不影响 NWD，且默认关闭零影响 | 维护成本低；DDP 下需确认 patch 在每个进程都执行 |
| 用闭式 W₂² 而非 AABB 投影 | Ultralytics `_get_covariance_matrix` 已经返回旋转协方差，闭式解 O(N) 可用 | 精度更高，与 ProbIoU 算力同量级 |
| 沿用 Ultralytics 的 w²/12 协方差约定 | 与 ProbIoU 一致，避免双轨制 | C 经验值需要校准（论文 C=12.8 → 我们 C≈64-100） |
| 默认 `--use-nwd` 关闭 | 向后兼容，新参数不破坏现有训练命令 | 用户主动 opt-in；resume 时需手动重新加 `--use-nwd` |
| 烟雾测试不依赖 ultralytics（核心数学层） | 即使没装 ultralytics 也能验证数学 | 测试矩阵分层：math 层用本地 cov、integration 层用 ultralytics cov |

## 5. 风险与遗留问题

### 5.1 已知风险

1. **C 值选错** → NWD 永远接近 1 或 0，没有梯度。文档已有指南，但需训练时实际观察 box_loss 曲线。
2. **ultralytics 升级破坏 monkey-patch** → 若 `_get_covariance_matrix` 或 `RotatedBboxLoss.forward` 签名变化，需同步更新 `NWDRotatedBboxLoss`。已在文档第 8 节注明。
3. **DDP 多卡** → monkey-patch 在 spawn 模式下需要每个子进程都执行 `enable_nwd_loss()`。train.py 在 `main()` 入口调用，理论上 fork 后子进程继承，但 spawn 模式（Windows 默认）需要验证。

### 5.2 未解疑问

- DOTA 上 NWD 的真实增益幅度（论文在 AI-TOD 提了 ~9 mAP，但 DOTA 目标尺寸跨度更大）
- 加上 P2 头之后 NWD 是否还有边际收益（P2 已经从架构上补救小目标）
- α 是否需要训练过程中动态调整
