# 项目文档

本目录按 SOP 规范存放项目所有过程性文档，便于后续追溯与复现。

## 目录结构

```
docs/
├── README.md              # 本文件，说明文档体系
├── paper/                 # 文献阅读笔记（SOP 第 1 节）
├── design/                # 设计方案与集成思路（SOP 第 3 节前置）
├── code/                  # 代码开发文档（SOP 第 3 节）
├── experiment/            # 算法与训练实验记录（SOP 第 2 节）
├── progress/              # 项目阶段性进度文档（SOP 第 4 节）
└── templates/             # SOP 文档模板
```

## SOP 摘要

所有任务结束时必须留下可追溯、可复现的书面记录。

| 文档类型 | 目录 | 关键要素 |
|----------|------|----------|
| 文献阅读 | `paper/` | 基础信息、核心痛点、创新点、实验、启发落地 |
| 实验训练 | `experiment/` | 实验目的、环境配置、超参、结果、深度分析 |
| 代码开发 | `code/` | 需求逻辑、技术栈、使用说明、避坑指南 |
| 进度归档 | `progress/` | 已完成、未完成、下次推进点 |

## 命名规范

- 文件名小写 + 连字符：`nwd-loss.md`、`yolo-obb-loss-analysis.md`
- 进度文档加日期前缀：`2026-05-16-nwd-loss.md`
- 一个主题一份文档，避免单文件超过 500 行

## 文档生命周期

1. 任务开始前阅读相关历史文档（特别是 `progress/` 最新一份）
2. 任务进行中实时更新对应文档
3. 任务结束时必写一份 `progress/` 阶段性总结
